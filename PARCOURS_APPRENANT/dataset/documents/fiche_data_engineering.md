# Fiche Métier — Data Engineer

## Définition

Le **Data Engineer** est responsable de la conception, de la construction et de la maintenance des infrastructures permettant de transporter et de stocker les données à grande échelle. Il constitue le pont entre les données brutes et les cas d'usage analytiques ou ML.

## Responsabilités clés

- Concevoir et maintenir des pipelines de données robustes et scalables
- Garantir la qualité et l'intégrité des données
- Optimiser les performances des requêtes et des traitements
- Collaborer avec les Data Scientists pour fournir les données nécessaires

## Compétences techniques

### Compétences obligatoires

| Compétence | Niveau attendu | Outils/Technos |
|------------|----------------|----------------|
| Python | Avancé | pandas, PySpark, scripts |
| SQL | Avancé | PostgreSQL, BigQuery, Snowflake |
| Orchestration | Intermédiaire | Apache Airflow, Dagster |
| Stockage | Intermédiaire | S3, GCS, Delta Lake |
| Containers | Intermédiaire | Docker, Docker Compose |

### Compétences appréciées

- Spark / PySpark pour le traitement distribué
- Kafka ou Pulsar pour le streaming
- dbt pour la transformation SQL
- Terraform pour l'IaC

## Parcours type vers le métier

### Profil idéal de recrutement

- Développeur Python avec 2-3 ans d'expérience
- Connaissance de SQL et des bases de données
- Notions de cloud (AWS, GCP ou Azure)

### Parcours de formation (18-24 mois)

| Étape | Formation | Durée |
|-------|-----------|-------|
| 1 | Python pour la Data | 24h |
| 2 | SQL Avancé et Optimisation | 16h |
| 3 | Data Engineering avec Apache Spark | 40h |
| 4 | Airflow et Orchestration | 16h |
| 5 | Fondamentaux DevOps pour Data | 16h |

**Total** : ~112 heures

## Évolution possible

- **Data Architect** : Conception de l'architecture data globale
- **Analytics Engineer** : Transformation SQL avec dbt
- **Platform Engineer** : Infrastructure data interne
